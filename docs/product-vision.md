# ConnectBloom Product Vision

## ピボット方針

ConnectBloom は、恋愛・交際を主目的にしたサービスではなく、**紹介制・招待制のコネクト SNS** へピボットします。

新しいコンセプトは次の通りです。

> 紹介や共通の興味をきっかけに、一緒に何かを楽しむ人・作る人・話せる人とつながる、招待制コネクトSNS

ConnectBloom は、信頼できる紹介や共通の活動テーマを通じて、人と人が安心してつながれる場所を目指します。

## 提供価値

- 信頼できる紹介から、活動仲間とつながる。
- 共通の興味から、ゆっくり会話が始まる。
- 一緒に作る、話す、出かける、学ぶ人と出会う。
- 大量スワイプではなく、少人数のプロフィールを丁寧に読む。
- 招待コードを通じて「誰のご縁から来たか」を記録する。

## プロフィールの軸

プロフィールは活動・興味・共創ベースに寄せます。

- 表示名
- 年齢
- 活動エリア
- つながり方のスタンス
- 活動ジャンル
- 興味のあること
- できること
- 一緒にやりたいこと
- 話したいテーマ
- 自己紹介

既存 DB の `location` は活動エリアとして使い、`dating_temperature` は UI 上「つながり方のスタンス」として扱います。内部名は当面維持します。

## 法務・安全方針メモ

ConnectBloom の主目的は、活動・興味・共創・紹介によるつながりです。以下に寄せない方針を明確にします。

- 恋愛・交際を主目的にしない。
- 異性紹介を目的にしない。
- 性別で相手を探す設計にしない。
- 恋愛対象や結婚願望の項目を置かない。
- サービスの主目的は、活動・興味・共創・紹介によるつながりである。

ただし、法的判断は別途専門家確認が必要です。公開・広告・ストア掲載・利用規約整備の前に、専門家へ確認してください。

## 募集ボード（主力機能）

ConnectBloomの次の主力機能として、ユーザーが「一緒にやりたいこと」「探している仲間」「参加してほしい活動」を投稿できる **募集ボード** を追加する。これは恋愛目的ではなく、活動・興味・共創・イベント・趣味仲間を軸に、人が自然に集まるための場である。

第一弾では、募集投稿の作成、一覧表示、詳細表示、参加したいボタンを提供する。カテゴリや興味タグ、活動エリア、オンライン/オフラインの希望を添えることで、紹介から広がるつながりを「小さく始める活動」へ接続する。

次フェーズでは、投稿者による参加希望者管理、承認フロー、会話連携へ進める。募集から自動的にコネクトやDMを作るのではなく、安心して共創に進める導線を段階的に設計する。

### 募集ボード Phase 2: 参加希望者管理

Phase 2では、募集投稿者が自分の募集に届いた「参加したい」を管理できるようにする。投稿者は参加希望者の表示名、年齢、活動エリア、つながり方のスタンス、活動ジャンル / 興味タグ、参加希望メッセージ、参加希望日時、ステータスを確認し、参加希望中の人を承認または見送りできる。

この管理機能は、活動・興味・共創・イベント・趣味仲間を安心して次の段階へ進めるためのものである。承認後の会話連携は次フェーズ予定で、Phase 2では会話連携前の管理に留める。

### 募集ボード Phase 3: 募集管理と参加管理

募集ボード管理機能として、投稿者向けの `/my-board` と参加希望者向けの `/my-interests` を追加する。投稿者は自分の募集を一覧で確認し、ステータス、参加希望数、承認済み数を見ながら、詳細確認、参加希望者管理、締切、再開、削除またはアーカイブを行える。

投稿者は募集詳細の「参加希望者」セクションで、届いた参加希望を確認し、参加希望中の人を承認 / 見送りできる。参加希望者は、自分が送った参加希望を一覧で確認し、必要に応じて取り消せる。

承認後の会話導線は、既存DMへ安全に接続できる場合は接続し、接続条件が整理できない場合は次フェーズで導線を設計する。募集ボードは、引き続き活動・興味・共創・紹介から広がるつながりを支える管理体験として育てる。
