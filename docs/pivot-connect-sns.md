# Pivot: 紹介制コネクト SNS

## 目的

ConnectBloom を、恋愛目的の体験から、活動・興味・共創・紹介を軸にしたコネクト SNS へ方向転換します。

今回の第一弾では、DB テーブル名や RPC 名を大規模に変えず、UI 文言・プロフィール項目・タグ・発見軸・docs・軽い色味調整を優先します。

## 新しい世界観

ConnectBloom は「誰かを選別する場所」ではなく、「紹介や共通テーマをきっかけに、話せる人・作れる人・一緒に活動できる人とつながる場所」です。

## 募集ボードによる共創導線

コネクトSNSへのピボット後の主力体験として、**募集ボード** を追加する。ユーザーは、AIアプリ制作、ブログ作業会、音声配信、映画やイベント同行、ダンス練習、ゲーム制作、海外や地域交流など、活動・興味・共創を軸にした募集を投稿できる。

Phase 1では、募集の投稿・一覧・詳細・参加したいを実装する。これにより、プロフィールだけで人を探すのではなく、「小さく始める活動」から紹介制のつながりが広がる体験を作る。

次フェーズでは、参加希望者管理、承認フロー、会話連携を追加し、募集をきっかけに安心して共創や活動仲間づくりへ進める導線を強化する。

## 募集ボード Phase 2: 参加希望者管理

募集ボードのPhase 2では、投稿者が自分の募集に届いた参加希望を確認し、承認または見送りできる管理導線を追加する。確認項目は、表示名、年齢、活動エリア、つながり方のスタンス、活動ジャンル / 興味タグ、参加希望メッセージ、参加希望日時、ステータスを中心にする。

承認済みの参加希望は、次フェーズで会話につなげられる予定とする。今回は会話連携前の管理フェーズであり、募集から自動でコネクト作成やDM作成は行わない。活動・興味・共創を軸に、投稿者が安心して次に進める人を整理するための機能として位置づける。

## 募集ボード管理機能

募集ボードを実用的な活動・共創の入口にするため、投稿者向けの募集管理と参加希望者向けの参加管理を追加する。投稿者は `/my-board` で自分の募集を確認し、締切、再開、削除またはアーカイブを行える。募集詳細では、参加希望者を確認し、参加希望中の人を承認 / 見送りできる。

参加希望者は `/my-interests` で自分が送った参加希望を確認し、参加希望中、承認済み、見送り、取り消し済みの状態を把握できる。参加希望の取り消しもこの導線から行う。

承認後の会話導線は、既存DMに安全に接続できる場合は接続し、条件整理が必要な場合は次フェーズで扱う。募集ボード管理は、紹介から広がるつながりを「一緒にやりたいこと」へ進めるための運用基盤として位置づける。

## 募集ボード編集機能

募集ボードでは、投稿者が投稿後に募集内容を編集できるようにする。編集できるのは投稿者本人またはadminのみで、参加希望が届いている募集には、内容が大きく変わりすぎないよう注意文言を表示する。

編集対象は、一緒にやりたいこと、探している仲間、活動ジャンル、活動エリア、興味タグ、募集人数、開催予定日、オンライン/オフライン形式、募集ステータスを中心にする。締切 / 再開 / 削除 / 編集が一通り管理できることで、募集ボードを活動・興味・共創の運用基盤として育てる。

## ルームの位置づけ

ルームは、紹介制・共創型Connect SNSとしてのConnectBloomにおける「会話先行」の入口です。募集ボードは目的や募集条件が固まった段階で使う導線ですが、ルームでは趣味・日常・制作アイデアなどの軽い会話を通じて、自然に企画やイベントの種が生まれる状態を目指します。

Phase 1では公式ルームを「クリエイティブルーム」と「雑談ルーム」に限定します。ユーザーはまず安心して話し、必要になったタイミングで募集ボードに接続します。この導線により、DMやマッチングより手前の低圧な交流を増やし、共創につながるきっかけを作ります。

## 通知センター Phase 1

コネクトSNSへのピボット後は、「参加希望が届いた」「承認された」「DMが届いた」ことに自然に気づける導線が重要になる。通知センター Phase 1では、アプリ内通知としてこの3種類に対応し、ユーザーが進行中のつながりを見失わないようにする。

通知はまずSupabaseから取得する通常の一覧表示に留め、リアルタイム通知やプッシュ通知は導入しない。将来的には通知設定、メール通知、PWA/iOS通知などへ拡張できる余地を残しながら、Phase 1ではConnectBloomらしい黄色×やさしい青のトーンで、安心感のある反応確認体験を整える。

## 通知センター Phase 1.5 / マイアクティビティ

- 通知センター Phase 1.5 では、未読通知件数・通知タイプラベル・既読導線を強化し、参加希望・承認・メッセージの反応に気づきやすくする。
- `/my-activity` を追加し、通知・自分の募集・参加希望した募集・会話/コネクト・ルームへの導線をまとめて確認できる場所にする。
- `/home` と `/settings` から通知・募集・参加希望・会話導線へ迷わず移動できるようにし、活動管理をスマホ表示で見やすいカードUIに整理する。
- プッシュ通知、メール通知、リアルタイム通知、通知設定の細分化、ルームメッセージ通知は今後の拡張として扱う。
